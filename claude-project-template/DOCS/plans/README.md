# plans/ — ordered implementation plans

Before building anything non-trivial, write the plan here first. One file per plan:

```
YYYY-MM-DD-REQ-XXX-short-title.md
```

A plan lists the steps in order, the acceptance criteria for each, and the evidence level each
step must reach. The "active plan" is the one currently being executed — `CLAUDE.md` tells
Claude to read it at the start of every session.

Keep finished plans here too (append-only). They're the record of how the work was structured.
